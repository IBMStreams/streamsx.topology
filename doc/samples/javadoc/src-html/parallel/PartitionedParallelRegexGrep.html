045/**
046 * PartitionedParallelRegexGrep is like ParallelRegexGrep, except that the Java
047 * object in the tuple being passed into the parallel region implements the
048 * Keyable interface, and provides a getKey() function which is used to map
049 * tuples to their corresponding channel in the parallel region.
050 * 
051 * Each channel of the parallel region only receives tuples that have the same
052 * hashCode() value of the Key returned by the tuple value's getKey() method. In
053 * other words, for each tuple, the value returned by
054 * 
055 * tupleValue.getKey().hashCode()
056 * 
057 * will go to the same channel, for each tuple which returns that result. To
058 * show this, instead of passing a java.lang.String into the parallel region, a
059 * stringWrapper class is created that implements the Keyable interface.
060 * 
061 * For this sample, if you read from a file that contains the following:
062 * 
063 * Apple Orange Banana Banana Apple Apple
064 * 
065 * you notice that the lines containing Apple will always be sent to the same
066 * channel of the parallel region; the same for the lines containing Orange and
067 * Banana.
068 * 
069 * 
070 */

078        String directory = args[1];

104                        // "apple" being sent to any other channel.
